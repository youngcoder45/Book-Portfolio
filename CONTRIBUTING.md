# Contributing Guidelines

Thanks for your interest in contributing to this 3D portfolio book project!

Because this repository is primarily a **personal portfolio**, contributions are limited to improvements that don’t change my personal identity, projects, or branding.

## What You Can Help With

- Fixing bugs in HTML/CSS/JS (layout issues, typos, broken links).
- Improving accessibility (ARIA labels, contrast, keyboard navigation).
- Performance optimizations (smaller assets, lighter animations).
- Non-breaking refactors that make the code easier to read and maintain.

## What Not to Change

- My name, bio, or personal story.
- My social links, email, or profile URLs.
- The core content of featured projects (names and links).

If you fork this project for your own portfolio, please:

- Replace all personal data (name, bio, links, images, email) with your own.
- Update the README to describe your version.

## Workflow

1. **Fork** the repository.
2. Create a new branch for your change:
   - `git checkout -b fix/layout-issue` or `feature/improve-accessibility`.
3. Make your edits in the `Book portfolio website` folder:
   - `index.html`
   - `style.css`
   - `script.js`
4. Test locally by opening `index.html` in your browser.
5. Commit your changes with a clear message:
   - `git commit -m "Fix mobile layout on skills page"`
6. Push your branch and open a **Pull Request** with a short description of:
   - What you changed.
   - Why it improves the project.

## Code Style

- Keep the existing HTML structure and classes where possible.
- Use consistent indentation and formatting to match the current files.
- Prefer semantic HTML where it doesn’t break the current design.

## Questions

If you have questions or suggestions before opening a PR, feel free to reach out via:

- Email: `contact@aditya-verma.me`

Thanks again for helping improve this project!
